# Workflow

## 1. 明确写作约束

- 目标：说服、汇报、归档、教学、提案
- 读者：管理层、同行、客户、评审、团队成员
- 输出介质：Markdown / Typst
- 证据：数据、引用、案例、附录

## 2. 先定结构

先用一句话回答：

> 读者读完后应该相信什么、记住什么、采取什么行动？

然后再拆成章节。

## 3. 大纲检查

- 是否结论先行
- 是否有清晰逻辑链
- 是否每节都有 `So What`
- 是否缺少关键证据

## 4. 写作与审阅

- 先写主结论和章节摘要，再补细节
- 初稿完成后按需委托 Oracle 做读者视角审阅
- Typst 只负责实现，不负责改写结构

## 5. 输出选择

| 场景 | 建议 |
| --- | --- |
| 快速协作、内部文档 | Markdown |
| 正式报告、论文、PDF | Typst |
